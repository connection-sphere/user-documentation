# Introduction

A **Reference Workflow** is a predefined workflow template that defines the structure, rules, branches, and actions for a campaign.

It acts as a reusable blueprint that multiple campaigns can follow without redefining the logic from scratch.

## Purpose

Reference Workflows allow:

* Standardization of outreach and automation strategies
* Reuse of proven workflow structures
* Faster campaign setup
* Easier maintenance and iteration

Instead of building workflows manually for each campaign, the system relies on reference workflows as the source of truth.

## Adaptive Nature

Reference Workflows are **adaptive by design**.

A campaign is not permanently bound to a single workflow. Instead, it can:

* Switch from one reference workflow to another
* Evolve its behavior over time
* Adjust to new strategies or conditions

This means the workflow system is not static — it supports dynamic transitions between different workflow definitions.
